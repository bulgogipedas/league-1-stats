# Liga 1 Analytics Dashboard

Professional football analytics dashboard for Liga 1 Indonesia, built with React, Vite, D3, Recharts, Tailwind CSS, Zustand, and an optional Bun/PostgreSQL backend.

The app is designed as a static-first analytics product: by default it reads preprocessed JSON from `public/data`. A local API can be enabled for PostgreSQL-backed development by setting `VITE_API_BASE_URL`.

## Current Status

- Frontend: production build passes with Vite.
- Data mode: static JSON by default, generated by `preprocess.py`.
- Backend: optional local Bun API with PostgreSQL support.
- Data freshness: available match files run until `15 May 2026`.
- AI analysis: automatic per selection. Uses Hugging Face only when `VITE_HF_TOKEN` is configured, otherwise local fallback text is shown.
- Logos: remote links from Football Logos are stored in `src/data/teamLogos.js`.

## Tech Stack

- React with Vite
- React Router v6
- D3.js for custom visualizations
- Recharts for simpler line charts
- Tailwind CSS
- Zustand for central data loading state
- Hugging Face Inference API, Qwen/Qwen2.5-72B-Instruct
- Optional Bun API and PostgreSQL

## App Pages

- `/overview`: standings, league summary, top performers, goal timeline, venue splits, rank timeline
- `/matches`: fixture selection, match comparison, xG timeline, passing trend, radar, result distribution
- `/players`: player search, rankings, profile, role percentiles, recommended comparisons, scout analysis
- `/teams`: team overview, leaders, best XI, trends, profile charts, squad links
- `/formations`: formation usage, pitch proxy, tactical outcomes, pressing and crossing trends

## Data Pipeline

Run preprocessing after changing any raw CSV:

```bash
npm run preprocess
```

The script:

- Reads raw CSV files from `player`, `match analisis`, `formation`, and `team analisis`.
- Parses scorelines into `goals_for` and `goals_against`.
- Calculates derived metrics such as pass accuracy, points, xG difference, conversion rate, and estimated player totals.
- Merges player per-90 metrics with performance scores.
- Generates JSON into `public/data`.
- Syncs standings against the official Liga Indonesia Baru snapshot embedded in `preprocess.py`.

Important data notes:

- Player file `player-metrics per 90 minutes.csv` is a rate table. Goals and assists in the UI are estimated totals, calculated from per-90 value multiplied by minutes divided by 90.
- Match-level charts still use local CSV match files.
- Formation pitch positions are tactical proxies, not tracking data.
- Attacking direction, set pieces, and pressing are proxy metrics based on available columns.

## Optional Backend

The frontend does not require the backend in production. Use it only when you want PostgreSQL-backed development.

Start PostgreSQL:

```bash
npm run db:up
```

Seed the database:

```bash
npm run preprocess
npm run db:seed
```

Start the API:

```bash
npm run api:dev
```

Set this in `.env` to make the frontend try the API first:

```text
VITE_API_BASE_URL=http://127.0.0.1:8787/api
```

If the API is unavailable, the frontend falls back to static JSON.

## AI Analysis

Set a Hugging Face token to enable live Qwen analysis:

```text
VITE_HF_TOKEN=your_token_here
```

Without the token, the app shows deterministic fallback analysis generated from the current data selection. The UI does not require a manual refresh button because analysis updates automatically when the selected data changes.

## Deployment Notes

Recommended static deployment:

1. Run `npm run preprocess`.
2. Run `npm run build`.
3. Deploy `dist`.
4. Leave `VITE_API_BASE_URL` unset unless an API deployment exists.
5. Configure `VITE_HF_TOKEN` only if live AI analysis is required.
